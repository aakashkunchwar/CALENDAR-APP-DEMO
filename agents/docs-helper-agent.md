Scan all source files in this project (index.html, style.css, app.js) and generate (or overwrite) README.md.

The README must be beginner-friendly and include:

1. **Project title & one-line description**
2. **Features** — bullet list of everything the app can do
3. **Getting Started** — step-by-step setup (clone repo, open index.html in a browser, no install needed)
4. **How to Use** — example walkthrough: adding an event, editing it, deleting it, navigating months
5. **Project Structure** — file tree with a one-line description per file
6. **Tech Stack** — simple table (HTML / CSS / JS / localStorage)
7. **Accessibility** — plain-English summary that the app meets WCAG 2.1 AA

Keep language simple. No jargon. Assume the reader has never used a calendar app or opened dev tools.
